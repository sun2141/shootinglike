# 프리킥 영상 분석 앱 설계안

## 1. 핵심 결론

공 속도 분석은 포함할 수 있다. 다만 초기 버전에서는 "정밀 계측"이 아니라 "조건부 추정"으로 설계해야 한다.

현실적인 제품 방향은 다음과 같다.

- 원본 영상은 기본적으로 저장하지 않는다.
- 사용자 기기에서 가능한 분석은 최대한 온디바이스로 처리한다.
- 서버에는 영상이 아니라 키포인트, 공 좌표, 분석 점수, 사용자 동의 기반 익명 데이터만 저장한다.
- 공 속도는 `평균 공속도`, `초기 추정 공속도`, `신뢰도 등급`으로 표시한다.
- 공개 데이터셋은 초기 기준 DB 구축과 모델/지표 설계에 사용하고, 장기적으로는 사용자 동의 기반 자체 DB를 만든다.

## 2. 공속도 분석 가능성

### 2.1 가능한 방식

공속도는 크게 두 방식으로 추정할 수 있다.

1. 비행 시간 기반 평균 속도
   - 사용자가 프리킥 거리, 예: 18m, 22m, 25m를 입력한다.
   - 영상에서 임팩트 시점과 골대/벽/착지 시점을 찾는다.
   - `평균 속도 = 이동 거리 / 비행 시간`으로 계산한다.
   - 장점: 구현이 비교적 쉽고 비용이 낮다.
   - 단점: 공 궤적이 휘거나 카메라에 끝 지점이 안 보이면 오차가 커진다.

2. 프레임 이동량 기반 속도
   - 공을 프레임별로 추적한다.
   - 골대 폭 7.32m, 공 지름 약 22cm, 페널티 라인, 골라인 등으로 픽셀-미터 변환을 한다.
   - 임팩트 직후 0.1-0.4초 구간의 공 위치 변화를 이용해 속도를 추정한다.
   - 장점: 순간 속도에 더 가깝다.
   - 단점: 카메라 각도, 원근, 모션블러, 낮은 FPS에 민감하다.

초기 앱에서는 두 방식을 함께 사용한다.

- `평균 공속도`: 거리와 비행 시간 기반
- `초기 공속도 추정`: 임팩트 직후 공 추적 기반
- `신뢰도`: 촬영 조건, FPS, 공 검출 품질, 골대/필드 보정 품질로 산정

### 2.2 정확도 조건

공속도 정확도는 촬영 조건에 크게 좌우된다.

| 조건 | 권장 수준 | 영향 |
|---|---:|---|
| FPS | 60fps 이상, 가능하면 120fps | 임팩트 직후 공 이동을 잡기 쉬움 |
| 해상도 | 1080p 이상 | 공 검출 안정성 증가 |
| 촬영 각도 | 골대와 키커가 모두 보이는 후방 30-45도 또는 측면 | 거리/방향 추정 오차 감소 |
| 영상 길이 | 임팩트 전 1초, 임팩트 후 2-3초 | 도움닫기, 임팩트, 결과 분석 가능 |
| 입력값 | 프리킥 거리, 골대 보임 여부 | 평균 속도 보정에 필요 |
| 공 가시성 | 임팩트 후 최소 5-10프레임 이상 | 초기 속도 추정 가능 |

### 2.3 표시 방식

정확한 단일 수치처럼 보이면 신뢰 문제가 생긴다. 다음처럼 표시한다.

- 평균 공속도: `82 km/h`
- 추정 범위: `76-88 km/h`
- 신뢰도: `보통`
- 이유: `60fps, 공 추적 8프레임 성공, 골대 기준 보정 성공`

신뢰도가 낮을 때는 다음처럼 표시한다.

- `공이 임팩트 직후 프레임에서 흐려져 속도 신뢰도가 낮습니다. 120fps 또는 더 밝은 환경에서 촬영하면 정확도가 올라갑니다.`

## 3. 공개 데이터셋 활용 전략

### 3.1 상업 사용 관점

| 데이터셋 | 활용 판단 | 사용 방식 |
|---|---|---|
| SoccerKicks | 사용 가능성 높음 | CC0-1.0. 데드볼 킥 자세 기준 DB의 초기 씨앗으로 사용 |
| Wyscout Soccer Match Event Dataset | 사용 가능 | CC BY 4.0. 프리킥 위치, 이벤트, 결과 기준 구축 |
| IDSSE Sportec Open Data | 사용 가능 | CC BY 4.0. 선수/공 위치와 이벤트 동기화 기준 참고 |
| 3D Shot Posture Dataset | 제한적 사용 | Apache-2.0이나 SoccerNet 파생 데이터라 직접 제품 DB 반영 전 검토 |
| SoccerNet | 주의 | 영상 다운로드는 NDA 필요. 상업 제품 DB보다는 연구/벤치마크 참고 |
| StatsBomb Open Data | 주의 | 무료지만 비상업 성격이 강함. 상업 앱 핵심 DB에는 비권장 |

### 3.2 초기 기준 DB

초기 DB는 원본 영상을 많이 저장하는 방식이 아니라, 분석에 필요한 파생 지표 중심으로 구축한다.

#### 자세 기준 DB

출처: SoccerKicks, 3D Shot Posture Dataset 참고

저장 지표:

- 도움닫기 각도
- 마지막 3보 보폭 변화
- 임팩트 직전 무릎 각도
- 디딤발-공 거리
- 디딤발 방향
- 골반 회전량
- 상체 기울기
- 팔 위치와 균형
- 임팩트 후 팔로스루 방향

#### 상황/결과 기준 DB

출처: Wyscout, IDSSE

저장 지표:

- 프리킥 거리
- 좌우 각도
- 슛/크로스 구분
- 골문 방향
- 수비벽 위치 추정 가능 여부
- 결과: 골, 유효슈팅, 벽 맞음, 골문 이탈, 세이브
- 난이도 점수

#### 공속도 기준 DB

초기 공개 데이터만으로는 공속도 기준이 충분하지 않다. 따라서 다음 기준으로 자체 누적한다.

- 사용자가 입력한 거리
- 영상 FPS
- 임팩트 프레임
- 도착/차단/착지 프레임
- 평균 공속도
- 초기 추정 공속도
- 검출 신뢰도
- 촬영 각도

## 4. 사용자 영상 저장 정책

### 4.1 기본 정책

원본 영상은 저장하지 않는다.

분석 후 저장하는 데이터:

- 사용자 ID 또는 익명 세션 ID
- 촬영 메타데이터: FPS, 해상도, 영상 길이
- 포즈 키포인트 JSON
- 공 위치 시계열
- 골대/필드 기준점
- 계산된 분석 지표
- 리포트 결과
- 사용자가 직접 입력한 결과 라벨

저장하지 않는 데이터:

- 원본 영상
- 얼굴 이미지
- 오디오
- GPS 위치
- 불필요한 장치 식별자

### 4.2 선택적 저장

모델 개선을 위해 원본 영상 저장이 필요할 수 있다. 이 경우 기본값은 꺼둔다.

동의 문구 예시:

> 분석 정확도 개선을 위해 이 영상을 익명화하여 저장할 수 있습니다. 동의하지 않아도 분석 기능은 사용할 수 있습니다.

보관 정책:

- 미동의: 원본 즉시 삭제
- 동의: 최대 30일 또는 90일 보관
- 사용자가 언제든 삭제 요청 가능
- 모델 학습 반영 전 얼굴/오디오 제거 또는 비식별 처리

## 5. MVP 제품 설계

### 5.1 사용자 흐름

1. 촬영 가이드 표시
2. 사용자가 5-10초 영상 업로드 또는 촬영
3. 프리킥 거리 입력
4. 분석 시작
5. 임팩트 순간 자동 탐지
6. 자세, 공 궤적, 공속도 추정
7. 결과 리포트 제공
8. 사용자가 결과 라벨 확인
9. 공유용 이미지 또는 짧은 리포트 생성

### 5.2 촬영 가이드

앱 첫 화면에서 복잡한 설명 대신 촬영 성공률을 높이는 짧은 가이드를 제공한다.

- 키커 전신과 공이 보이게 촬영
- 골대가 보이면 공속도 정확도 상승
- 60fps 이상 권장
- 임팩트 전 1초부터 임팩트 후 3초까지 촬영
- 가능하면 삼각대 또는 고정 촬영
- 거리 입력: 18m, 20m, 22m, 25m, 직접 입력

### 5.3 분석 결과 화면

초기 반응을 만들기 위해 결과는 숫자보다 직관성이 중요하다.

주요 카드:

- 종합 프리킥 점수
- 공속도
- 킥 타입
- 정확도
- 임팩트 품질
- 디딤발 안정성
- 상체 밸런스
- 팔로스루
- 개선 포인트 3개

예시:

```text
프리킥 점수: 78
공속도: 84 km/h (신뢰도 보통)
킥 타입: 인스텝 파워형

좋은 점
- 임팩트 직전 무릎 각도가 안정적입니다.
- 팔로스루가 목표 방향으로 이어집니다.

개선 포인트
- 디딤발이 공보다 약간 뒤에 있어 공이 뜰 가능성이 있습니다.
- 상체가 임팩트 직전 뒤로 젖혀집니다.
- 공을 차는 순간 발목 고정이 약합니다.
```

### 5.4 공유 기능

초기 바이럴을 위해 공유용 결과물을 만든다.

- 분석 점수 카드
- 공속도 카드
- 킥 타입 카드
- 골문 9분할 타깃맵
- 임팩트 순간 스틸 이미지
- 3초 하이라이트 오버레이 영상

원본 영상을 저장하지 않는 정책과 충돌하지 않도록, 공유용 결과물은 사용자의 기기에서 생성하고 사용자가 직접 저장/공유한다.

## 6. 분석 엔진 설계

### 6.1 입력

- 영상 파일: mp4, mov
- 권장 길이: 5-10초
- 권장 FPS: 60fps 이상
- 사용자 입력:
  - 프리킥 거리
  - 주발
  - 목표: 골문, 크로스, 벽 넘기기
  - 결과: 골, 유효슈팅, 벽 맞음, 빗나감

### 6.2 처리 단계

1. 영상 메타데이터 추출
   - FPS
   - 해상도
   - 길이

2. 사람 포즈 추출
   - MediaPipe Pose 또는 YOLO Pose 사용
   - 어깨, 골반, 무릎, 발목, 발끝 좌표 추출

3. 공 검출
   - 초기에는 경량 YOLO ball detector 사용
   - 실패 시 사용자가 공 위치를 2-3프레임에서 직접 지정하는 보정 UI 제공

4. 임팩트 탐지
   - 발과 공의 최소 거리
   - 공 속도 급증 프레임
   - 발목/발끝 속도 피크

5. 골대/필드 보정
   - 골대가 보이면 골대 폭 7.32m 기준
   - 골대가 안 보이면 공 지름, 사용자 입력 거리, 발-공 관계 기반 보정

6. 공속도 추정
   - 평균 공속도: 거리 / 비행 시간
   - 초기 추정 공속도: 임팩트 직후 공 좌표 변화량
   - 두 값을 신뢰도에 따라 가중 평균

7. 리포트 생성
   - 점수화
   - 개선 포인트 생성
   - 공유 카드 생성

### 6.3 공속도 계산 로직

#### 평균 공속도

```text
distance_m = user_input_distance_m or calibrated_distance_m
flight_time_s = arrival_frame - impact_frame / fps
avg_speed_mps = distance_m / flight_time_s
avg_speed_kmh = avg_speed_mps * 3.6
```

#### 초기 추정 공속도

```text
ball_positions = tracked ball coordinates after impact
scale_m_per_px = calibration result
delta_px = distance between ball positions across N frames
delta_m = delta_px * scale_m_per_px
delta_t = frame_count / fps
initial_speed_kmh = delta_m / delta_t * 3.6
```

#### 신뢰도 산정

```text
confidence =
  fps_score
  + ball_visibility_score
  + calibration_score
  + camera_angle_score
  + tracking_stability_score
```

등급:

- 높음: 80점 이상
- 보통: 50-79점
- 낮음: 50점 미만

## 7. 비용 최소화 아키텍처

### 7.1 추천 구조

초기에는 모바일 앱보다 PWA 또는 모바일 웹으로 시작하는 것이 비용이 낮다.

구성:

- Frontend: Next.js 또는 React PWA
- 온디바이스 분석: TensorFlow.js, MediaPipe, WebAssembly
- Backend: Supabase 또는 Firebase 무료/저가 플랜
- Storage: 기본적으로 원본 영상 저장 없음
- DB: Postgres
- Auth: 익명 로그인 + 선택적 소셜 로그인
- Analytics: PostHog 또는 Firebase Analytics

## 9. 분석 기준 점수화

### 9.1 종합 점수

```text
overall_score =
  posture_score * 0.35
  + impact_score * 0.25
  + ball_speed_score * 0.15
  + accuracy_score * 0.15
  + consistency_score * 0.10
```

초기에는 정확도와 일관성 데이터가 부족하므로, 사용자 누적 데이터가 쌓이기 전까지는 자세와 임팩트 비중을 높인다.

### 9.2 공속도 점수

거리와 연령/성별/수준 정보를 받지 않는 초기 버전에서는 절대 평가보다 구간 평가가 적합하다.

예시:

| 공속도 | 표시 |
---:|---|
| 60km/h 미만 | 컨트롤 중심 |
| 60-80km/h | 안정적인 킥 |
| 80-100km/h | 강한 킥 |
| 100km/h 이상 | 매우 강한 킥 |

단, 유소년/아마추어 사용자가 많을 수 있으므로 "약하다" 같은 표현은 피한다.

## 12. 최종 제품 포지셔닝

초기 카피:

> 내 프리킥 영상만 올리면, 공속도와 자세 문제를 30초 안에 분석해주는 앱

초기 핵심 가치는 다음 세 가지다.

1. 공속도가 나온다.
2. 내 자세 문제가 눈에 보인다.
3. 바로 공유하고 다시 도전하고 싶다.

이 방향이면 고가 라이선스 영상 없이도 출시 가능하고, 초기 비용을 낮추면서 사용자 반응을 확인할 수 있다.
