# 🚀 우리만의 프리킥 전용 AI 만들기 (자가개발 가이드)

본 가이드는 사용자들이 자발적으로(Opt-in) 제공한 영상 데이터를 활용하여, 범용 모델이 아닌 **"우리의 앱 전용 초경량 볼 트래킹 모델"**을 비용 없이 구글 코랩(Colab)을 통해 훈련시키는 방법을 안내합니다.

## 1. 데이터 수집 확인
- 사용자가 체크박스에 동의 후 영상을 올리면, 영상과 프리킥 분석 결과값(Impact Frame 등)이 서버로 전송됩니다.
- 현재 MVP 버전에서는 `app/api/dataset/upload`를 통해 프로젝트 내 `.data/uploads` 폴더에 임시로 저장됩니다.

## 2. 데이터셋 구성 및 라벨링 (Roboflow 등 활용)
영상을 모은 뒤, 모델이 '공'이 무엇인지 알 수 있도록 네모(Bounding Box)를 그려주는 라벨링 작업이 필요합니다.
1. `roboflow.com` 에 가입 후 프로젝트를 생성합니다. (퍼블릭으로 생성 시 완전 무료)
2. `.data/uploads` 에 모인 영상(또는 영상에서 추출한 이미지 프레임)을 업로드합니다.
3. 영상 프레임마다 '공(Ball)' 부분에 박스를 치고 라벨링을 진행합니다.
4. 라벨링이 끝나면 `YOLOv8` 포맷으로 데이터셋(`dataset.zip`)을 내보내기(Export) 합니다.

## 3. 구글 코랩(Colab)에서 무료로 훈련하기
비싼 GPU 서버를 살 필요 없이 구글의 무료 자원을 이용합니다.
1. 프로젝트 폴더 안의 `scripts/train_yolo_colab.ipynb` 파일을 구글 드라이브에 업로드하고 코랩으로 엽니다.
2. 코랩 좌측 폴더 모양 아이콘을 눌러, 앞서 다운받은 `dataset.zip` 파일을 업로드합니다.
3. 주피터 노트북에 적힌 순서대로 `재생(▶)` 버튼을 누르면 자동으로 패키지를 설치하고 훈련을 시작합니다.
4. 보통 1~2시간 내에 훈련이 완료됩니다.

## 4. 앱에 커스텀 모델 적용하기 (TensorFlow.js)
훈련이 끝나면 앱(브라우저)에서 바로 돌아갈 수 있도록 파일이 변환됩니다.
1. 코랩 마지막 단계를 실행하면 `best_web_model` 이라는 폴더가 생성됩니다. (안에 `model.json`과 여러 `.bin` 파일들이 들어있음)
2. 이 파일들을 통째로 다운로드하여, 우리 프로젝트의 `public/model/` 폴더에 넣습니다.
3. `lib/workers/ball.worker.ts` 파일을 열고, 기존 COCO-SSD를 부르던 코드를 방금 생성한 커스텀 모델로 부르도록 수정합니다.

```typescript
// ball.worker.ts 변경 예시
import * as tf from '@tensorflow/tfjs';

// 기존: model = await cocoSsd.load();
// 변경:
model = await tf.loadGraphModel('/model/model.json');
```

위의 4단계 과정을 거치면, 점점 더 많은 사용자의 데이터를 학습하며 **어떤 환경에서도 정확하게 공을 쫓아가는 완벽한 프리킥 엔진**으로 진화하게 됩니다!
