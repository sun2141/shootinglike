# Phase 6 Plan — 2차 MVP 확장

## 목표
1차 MVP(Phase 1~5) 완료 후, 분석 정확도·재방문성·차별화를 높이는 기능 확장.

## 후보 기능 (우선순위 제안)

### A. 레퍼런스 DB 운영 고도화 (정확도/운영성)
- 외부 현장에서 레퍼런스 영상을 바로 등록할 수 있도록 private Blob/S3 저장소 연결.
- 원본 영상은 private storage에 두고, DB에는 `sourceUrl`/`blobPath`/파일 메타데이터/실제값/앱 분석값/보정값만 저장.
- Draft/Active/Rejected 상태, 이상치 factor 경고, CSV import/export, 관리자 토큰/권한 UX를 보강.
- 초기에는 global median 보정을 쓰되, 데이터가 쌓이면 촬영 각도/거리/FPS/기기/선수 키 조건별 calibration set으로 확장.

### B. 레퍼런스 세그먼트 워크플로우 검증 (정확도/생산성)
- `Run Code`로 Gemini/FFmpeg 명령을 붙여넣고 Manual Segments를 자동 생성하는 흐름을 실제 긴 원본 영상으로 검증.
- FFmpeg.wasm `cut clips`의 재인코딩 속도, 진행률, 실패 메시지, 브라우저별 메모리 한계를 점검.
- clip row 저장, 거리/속도 입력, PIXEL CAPTURE 측정, DB 저장까지 한 번에 이어지는 운영 절차를 문서화.

### C. Calibration 정확도 개선 (분석 품질)
- 공 검출 개선과 속도 스케일 보정을 분리해서 개선.
- 수동 좌표 입력은 일반 사용자용이 아니라 관리자 레퍼런스 보정 전용으로 유지.
- 골대 폭, 공 크기, 선수 키, 촬영 거리 같은 기준 길이 후보를 실험.
- 레퍼런스가 최소 5~10개 쌓이기 전에는 결과 신뢰도/과신 방지 문구를 유지.

### D. 리더보드 고도화 (재방문성)
- 거리별/사용자별 필터, 기간별 랭킹, 본인 순위 하이라이트.
- `/api/leaderboard` 쿼리 파라미터 확장 + `prisma` 쿼리 보강.

### E. 사용자 기록 비교 (재방문성)
- 동일 사용자의 과거 분석 대비 속도/자세 점수 추이 시각화.
- `User`/`Analysis` 모델 기반 개인 히스토리 페이지.

### F. 코치 모드 / 고급 분석 (차별화, 대형)
- 궤적 시뮬레이션, 관절별 상세 리포트 등. 범위가 크므로 후순위.

## 진행 규칙
- Next.js 코드 수정 전 `node_modules/next/dist/docs/` 관련 가이드 선확인 (AGENTS.md).
- 영상 원본 비저장 정책 유지. 보정 좌표 등 메타데이터만 처리.
- 원본 레퍼런스 영상을 영구 보관하려면 사용자/관리자 의도와 private storage 구성을 명확히 한 뒤 처리.
- 각 기능 착수 시 본 PLAN에 세부 작업 분해 추가.

## 상태
이전 Codex 스레드 맥락 이관 완료. 초기 레퍼런스 DB, PIXEL CAPTURE, Gemini/FFmpeg segment workflow는 구현되어 있으므로, 다음 착수 범위는 운영 고도화와 실제 영상 기반 검증 중에서 선택.
