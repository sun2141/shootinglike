# Phase 3: 분석 엔진 및 스코어링

## 목적 (Objective)
Phase 2에서 얻은 관절 픽셀 좌표와 임팩트 프레임을 기반으로, 유의미한 물리적 수치(km/h 속도, 자세 100점 만점)를 산출하고 사용자에게 구체적인 개선 텍스트(피드백)를 제공하는 로컬 분석 모듈을 구축합니다.

## 작업 목록 (Tasks)

1. **픽셀-미터 변환 (Pixel-to-Meter) 및 구속 추정 모듈 (`lib/analysis/speed.ts`)**
   - 랜드마크 0(코)과 27/28(발목) 간의 픽셀 거리를 계산하여 사용자의 표준 신장(기본값 1.75m)과 비례하는 배율(Scale)을 구합니다.
   - 임팩트 발생 프레임부터 이어지는 5~10 프레임 동안의 발끝(Foot Index) 혹은 포착된 픽셀의 이동 거리를 배율과 영상 fps(예: 60fps)를 이용해 물리적 속도(km/h)로 환산합니다.

2. **자세 점수 산출 모듈 (`lib/analysis/form.ts`)**
   - 임팩트 프레임 시점에서의 관절 각도를 계산하는 유틸리티 함수(Cosine Rule) 작성.
   - **디딤발 무릎 각도 (Plant Leg Knee Angle)**: 랜드마크 23/24(Hip) - 25/26(Knee) - 27/28(Ankle) 각도를 구합니다. (이상적 범위: 130~150도)
   - **상체 기울기 (Torso Lean)**: 랜드마크 11/12(Shoulder)와 23/24(Hip)의 수직선 대비 각도를 계산하여 밸런스를 측정.
   - 각도별 감점 요소를 합산하여 최종 'Form Score (0~100점)'를 계산합니다.

3. **로컬 룰셋 기반 피드백 텍스트 조합 (`lib/analysis/feedback.ts`)**
   - `form.ts`에서 계산된 오차(예: 디딤발이 너무 펴짐, 상체가 뜸)를 바탕으로 2~3개의 핵심 개선점 문장(Korean/English)을 하드코딩된 배열에서 매칭하여 조합.
   - 생성된 피드백 문구를 UI로 전달하기 위한 데이터 구조 생성.

4. **결과 UI 연동 (`app/analyze/page.tsx` 수정)**
   - 기존의 난수(Random) 점수와 속도를 실제 계산된 분석 결과와 피드백 문자열로 교체.

## 검증 방법 (Verification)
- 동일한 영상을 반복해서 넣었을 때 구속과 스코어가 일관성 있게 나오는지 확인.
- 계산된 각도(Angle)가 콘솔 로그 상에서 해부학적으로 타당하게 떨어지는지 로그로 확인.
- 피드백 문장이 정상적으로 UI에 표시되는지 체크.
